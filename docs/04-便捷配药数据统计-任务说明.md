## ✅ 图表 2: 新增医院便捷配药统计

**数据源**: SQLite 数据库 `business_flow.db`
- 表: `daily_flow_2026_jan_feb` + `daily_flow_2026_mar` + `daily_flow_2026_apr`
- 查询: UNION ALL 三个表，外层 GROUP BY date 聚合
- 医院: 齐鲁德医、齐鲁二院（齐鲁第二医院）、安徽省立、青岛中心
- 过滤: `ye_wu_lei_mu LIKE '%处方服务%'`, `pay_status='收费'`

**布局**: 右栏（与图表1并排）
- **顶部汇总**: 4 家医院总流水、总订单（2026 全年）
- **折线图**: 近 15 天，10 条线（4 家×流水+订单 + 总流水 + 总订单）
- **底部表格**: 近 15 日，每医院流水/订单行 + 总计行

**关键 SQL**:
```sql
SELECT SUBSTR(yewu_wancheng_shijian,1,10) as date, 1 as cnt, amount as amt
FROM {table}
WHERE institution LIKE '%{pattern}%' 
  AND ye_wu_lei_mu LIKE '%处方服务%' AND pay_status='收费'
  AND yewu_wancheng_shijian IS NOT NULL AND amount IS NOT NULL
-- 外层: SELECT date, SUM(cnt) as orders, SUM(amt) as flow FROM (...) GROUP BY date
```

**重要教训**:
- ❌ SQL 双聚合陷阱：内层子查询不要 `COUNT(*)/SUM()` + `GROUP BY`，否则外层再 `SUM()` 会导致累积值
- ✅ 正确做法：内层返回原始行（`1 as cnt, amount as amt`），聚合只在外层做一次
