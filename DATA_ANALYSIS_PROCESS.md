# 运营数据分析流程设计方案

## 🎯 目标

为运营团队提供可执行的数据分析流程，支持日常运营决策、问题发现和趋势预测。

## 📊 核心分析维度

### 1. 医院绩效分析
- **指标**：订单数、金额、客单价、复购率
- **维度**：医院、院区、时间（日/周/月）
- **用途**：评估业务健康度、识别高潜力医院

### 2. 业务类目分析
- **指标**：各类目订单分布、金额占比、增长趋势
- **重点类目**：处方服务（电子处方/便捷购药）、自营健管、第三方服务
- **用途**：优化资源配置、制定产品策略

### 3. 医院对比分析
- **指标**：同维度下各医院排名、差距分析
- **用途**：学习最佳实践、发现薄弱环节

### 4. 异常检测
- **指标**：环比/同比显著变化、异常低/高值
- **用途**：及时发现业务问题或机会

### 5. 时间序列分析
- **指标**：趋势、季节性、周期性
- **用途**：预测、排产、资源规划

## 📅 日常分析任务

### 每日监控（早上10点前）
- 各医院昨日订单数/金额
- 异常值预警（环比变化>30%）
- 处方服务关键指标

### 每周分析（周一）
- 上周各医院绩效排名
- 各类目_week-over-week变化
- 异常医院/类目专项分析

### 每月报告（月初5日前）
- 上月全量数据报告
- 关键指标趋势图（近3个月）
- 问题医院复盘
- 下月运营建议

## 🛠️ 数据分析流程（使用 Data Analysis 技能）

### 步骤1：明确决策问题
```
用户问：浙江中医院湖滨院区3月份业务怎么样？

转化为决策问题：
- 湖滨院区3月份业务健康度如何？
- 与2月份相比是增长还是下滑？
- 哪些类目增长快/慢，需要调整？
```

### 步骤2：定义指标合同
```
指标：处方服务订单数
- 实体：医院（湖滨院区）
- 粒度：日订单
- 分子：收退标识=收费 & 业绩类目 LIKE '%处方%'
- 分母：所有收费处方订单
- 过滤：排除黑龙江中医药大学附属第一医院等测试账号
- 时间：2025年3月1日-31日
- 数据源：daily_flow_2025 表
```

### 步骤3：数据提取与清洗
```
SQL查询：
SELECT 
    DATE(yewu_wancheng_shijian) as 日期,
    COUNT(*) as 订单数,
    SUM(amount) as 实际支付金额
FROM daily_flow_2025
WHERE institution LIKE '%浙江省中医院%'
  AND institution LIKE '%湖滨院区%'
  AND ye_wu_lei_mu LIKE '%处方服务%'
  AND pay_status = '收费'
  AND SUBSTR(yewu_wancheng_shijian, 6, 2) = '03'
GROUP BY DATE(yewu_wancheng_shijian)
ORDER BY 日期
```

### 步骤4：分析与可视化
- 折线图：时间序列趋势
- 柱状图：院内处方 vs 便捷购药对比
- 热力图：按星期几的订单分布

### 步骤5：决策输出（Decision Brief）
```
问题：湖滨院区3月业务健康度评估

答案：
- 订单数环比下降25%，需关注原因
- 便捷购药仍是主力（占91%）
- 院内处方业务稳定

证据：
- 3月订单14,035单，2月18,707单（-25%）
- 便捷购药12,966单（92%）
- 院内处方1,069单（8%）

置信度：高（数据完整，样本量充足）

注意事项：
- 需对比季节性因素
- 3月31日数据可能不完整

建议行动：
- 调查订单下降原因（市场/运营/竞品）
- 加强院内处方业务推广
- 启动促销活动挽回客户
```

## 📊 场景化分析示例

### 场景1：医院扩张评估
```
问题：哪5家医院适合重点拓展？

分析：
1. 按订单数排序（近3个月）
2. 按金额/GDP比值（潜力评估）
3. 按增长速度（高增长/high potential）

输出：排名列表 + 重点医院画像
```

### 场景2：业务优化
```
问题：便捷购药和院内处方哪个更重要？

分析：
- 各自的订单数、金额、客单价
- 增长趋势对比
- 医院覆盖率（多少医院做此类业务）

输出：两类目业务价值对比 + 优化建议
```

### 场景3：问题医院诊断
```
问题：某医院订单下滑了50%，为什么？

分析：
- 时间趋势（哪个月开始下滑）
- 类目结构变化（哪个类目下降最多）
- 同期对比（去年同期 vs 今年同期）
- 市场因素（竞品活动、政策变化）

输出：问题诊断报告 + 改进建议
```
